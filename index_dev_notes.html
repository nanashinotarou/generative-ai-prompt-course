01. UI Sounds (Web Audio API)

JavaScript AudioContext UX

「学習を単調な作業にしない」という目的のため、今回は音声ファイルを一切使わず、ブラウザ標準の Web Audio API (AudioContext) を用いて、プログラムから直接サウンド波形を生成しています。

MP3ファイルなどを読み込むオーバーヘッドがなく、アクションに対して完全に遅延ゼロで音が鳴るのが特徴です。「チッ」というサトルなクリック音、「ドスッ」というワックスシールのスタンプ音、完了時のチャイム音などをオシレーター（波形発生器）で合成しています。

// サウンド合成のコアロジック（抜粋） const now = audioCtx.currentTime; if (type === 'stamp') { // 低音の三角波で「ドスッ」という重みを表現 osc.type = 'triangle'; osc.frequency.setValueAtTime(150, now); osc.frequency.exponentialRampToValueAtTime(40, now + 0.15); gainNode.gain.setValueAtTime(0.2, now); gainNode.gain.exponentialRampToValueAtTime(0.01, now + 0.15); }

※ ユーザーが初めて画面をクリックしたタイミングで AudioContext を初期化することで、ブラウザの自動再生ポリシー（Autoplay Policy）を回避しています。

02. Sparkles (Canvas API)

HTML5 Canvas RequestAnimationFrame Physics Simulation

「魔法の体験」の視覚的コアとなるのが、コピーボタンをクリックした時や、調合釜にアイテムを入れた時に弾けるパーティクルエフェクトです。 画面全体を覆う透明な <canvas id="sparkles"> を pointer-events: none; で配置し、その上でJSによる物理演算を行っています。

// 重力と減衰のシミュレーション particles.forEach(p => { p.x += p.vx; p.y += p.vy; p.vy += 0.1; // Gravity (重力低下) p.alpha -= p.decay; // Fade out (透明化) });

この軽量なスクリプトにより、外部ライブラリ（Three.jsなど）に依存することなく、60FPSで滑らかに動作する美しいインタラクションを実現しました。

03. Prompt Cauldron (Drag & Drop UI)

HTML5 Drag and Drop Gamification CSS Animations

プロンプトの4要素（Role, Task, Context, Format）を学ぶ部分は、単なる箇条書きではなく「大釜（Cauldron）に材料を放り込む」というゲーミフィケーションUIを採用しました。

HTML5の標準Drag & Drop APIを使用し、dragstart, dragover, drop イベントをフックしています。アイテムが大釜の上に来た時（dragover）は釜が赤く光り、ドロップした瞬間にはCSS transform: scale(1.1) translateY(5px); で「ポチャッ」という沈み込みアニメーションと、前述のサウンド＆パーティクルが同時発動します。

04. Scenario Sandbox

Instructional Design Mockup UI

ページ後半の「業務への応用」セクションは、当初シンプルな箇条書きリストでしたが、「動画を見なくても100%内容が理解できる」「むしろテキストの方が充実している」という要件を満たすため、完全な Sandbox（砂場）UIへと大改修を行いました。
