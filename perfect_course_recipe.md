# 完璧な教科書のレシピ (Magical Experience v2.0)

このドキュメントは、今後の研修資料およびWebサイトを作成するAI（Gemini/Antigravity等）に対する「**究極のシステムプロンプト・設計ガイドライン**」です。
新しい研修ページを作成する際は、必ずこのドキュメントを読み込み、以下のUI/UX要件を完全に満たしたものを出力してください。

---

## 🎨 1. 根本的なUI/UXフィロソフィー

- **Magical Experience（魔法のような体験）**: 単なる「文字の羅列」ではなく、ユーザーが触れてワクワクするようなリッチな仕掛けを盛り込む。
- **Friendly Pro（親しみやすいプロ）**: 専門的で正しい知識を、極めて噛み砕いた「です・ます調」で、カラフルで楽しいトーンで伝える。
- **ネイティブスクロールの尊重**: スクロールジャックや、ユーザーの操作を奪う全画面ロック（Unlock Screen）などの「余計なギミック」は一律排除する。コンテンツは素直に下へ読めるようにする。
- **色彩の統一定義**:
  - メインテキスト: `#0f172a` (Slate 900)
  - サブテキスト: `#475569` (Slate 600)
  - アクセント（進行・達成）: `#10b981` (Emerald 500)
  - 警告・注意（Danger）: `#ef4444` (Red 500)
  - **クリッカブル（押せる合図）**: リンクやボタンには必ず `#0ea5e9` (Sky 500) 系統の色（または紫 `#8b5cf6`）を用い、「この色は押せる」という無意識のルールを作る。

## 🪄 2. 必須となる「究極のUIコンポーネント」

新しいページには、内容に応じて**以下のコンポーネントを必ず組み合わせて使用**してください。単なる `<p>` や `<ul>` の連続は禁止です。

### ① Glass Card Layout (メインコンテナ)
すべての章やセクションは、背景が透けるグラスモーフィズム調のカード（`.glass-card`）に格納する。
- 必須装飾: `backdrop-filter: blur(12px);`, わずかなボックスシャドウ、ホバー時の浮き上がり。

### ② Bento Grid (構成要素の分解表記)
4つ〜6つ程度の要素（プロンプトの構成要素、機能の紹介など）を並べる際は、Apple風の**Bento Grid layout** (`.bento-grid`, `.bento-item`) を使用する。
- 重要なアイテムには `grid-column: span 2` や `.card-magic` (暗背景＋星エフェクト) を適用して目立たせる。

### ③ Workflow Timeline (手順とプロセス)
思考のプロセスや操作手順（ステップ1, 2, 3...）は、単なる箇条書きではなく**水平方向のタイムライン**（`.workflow`, `.flow-step`）で表現する。
- アイコンを丸枠で囲み、ホバー時に少し跳ねるアニメーションをつける。

### ④ Browser Mockup (コードやGemini出力の臨場感)
コードブロックやAIへのプロンプト入力例、AIからの回答例を表示する際は、単なる黒背景の枠ではなく、**Macの偽ブラウザウィンドウ風UI**（`.browser-mockup`, 赤黄緑のドット付きヘッダー）の中に表示する。

### ⑤ Mission Ticket (実習課題の提示)
章の終わりには必ず「Mission（実習）」を設ける。この際、単なるチェックリストではなく、物理的な**チケット風デザイン**（`.mission-ticket`, `.ticket-stub`, 切り取り線の破線加工）にして、ユーザーの挑戦意欲を掻き立てる。

## 🎧 3. Magical Interactive Elements (魔法の仕掛け)

ユーザーの操作に対して、視覚と聴覚で「心地よい」フィードバックを返すこと。

### ① Web Audio APIによる極小サウンド効果
画面の右下に「ミュート解除ボタン」を浮遊させ、以下の操作時に心地よいサウンドを鳴らす（`playUISound('type')` 関数を実装すること）。
- タブ切り替え、チェックボックス操作時: カチッという極小の**クリック音**
- 課題（Mission）達成時: トスッという**スタンプ音**
- コードのCopyボタンクリック時: キラキラという**魔法の効果音**
- 全ミッション完了時: ポロロンという**クリアチャイム音**

### ② 物理スタンプ表現 (Wax Seal)
ミッションのチェックボックス（`.task-item`）をクリックして完了状態にした際、単にチェックマークが付くだけでなく、物理的な**「封蝋（ワックスシール）スタンプ」**が上からドンッとアニメーション付き（`@keyframes stampIn`）で押される演出を入れること。

### ③ 魔法の火花パーティクル (Sparkles)
コードをCopyした際や、UI上の特別なアクション（調合など）を行った際、カーソル位置を中心に金色の**火花パーティクル（Canvas描画）**を飛び散らせる。

### ④ ドラッグ＆ドロップ体験型シミュレーター
特に重要な概念（例：プロンプトの構成要素の組み合わせ、等）を教える際は、単なる読み物にするのではなく**「要素をドラッグして、釜（Cauldron）や特定の枠にドロップして完成させる」**体験型のJavaScriptシミュレーターを実装すること。

## 📚 4. 構成・コンテンツの型

ページ全体は以下の構成を基本とする：

1. **Hero Header**: 大きな見出し、DAYバッジ、本日の目標（Sub-goal）
2. **Fixed Progress Bar**: 画面上部に固定され、ミッション消化率（%）が伸びるプロンプトバー。100%達成でCanvas**紙吹雪（Confetti）**を降らせる。
3. **Tab Content (前半/後半など)**: コンテンツが長い場合はタブで分割し、スクロール疲れを防ぐ。
4. **動画と手厚い要約**: YouTube動画を埋め込む際はサムネイル画像風にしき、**必ずその右側や直下に「動画を見なくても内容が100%わかる手厚い要約（図解付き）」**を配置する。
5. **Mission (チケット UI)**: 章ごとの課題。チェックボックス連動。
6. **理解度チェック (Quiz Area)**: ページ最下部に、本日の内容のおさらいとなる簡単なクイズを設置する。
